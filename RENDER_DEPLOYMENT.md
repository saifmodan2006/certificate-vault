# Deploying CertiVault to Render

Complete guide to deploy both your backend and frontend to Render.

## Prerequisites

- GitHub account (repo already pushed there ✅)
- Render account (free at https://render.com)
- GitHub connected to Render

## Deployment Methods

### Method 1: Using render.yaml (Recommended) ⭐

**Simplest approach — uses included config file**

1. **Connect GitHub to Render**
   - Go to https://dashboard.render.com
   - Click "New +" → "Root Directory"
   - Connect your GitHub account
   - Select `saifmodan2006/certificate-vault`

2. **Deploy**
   - Render automatically detects `render.yaml`
   - Creates both backend and frontend services
   - Deploys both automatically

3. **Services Created**
   - **Backend:** `certivault-backend.onrender.com`
   - **Frontend:** `certivault-frontend.onrender.com`
   - Frontend automatically connects to backend via `NEXT_PUBLIC_API_URL`

---

### Method 2: Manual Setup (Alternative)

If you prefer setting up services individually:

#### Backend Deploy

1. Create new Web Service on Render
2. **Name:** `certivault-backend`
3. **Environment:** Python
4. **Build Command:** 
   ```
   cd backend && pip install -r requirements.txt
   ```
5. **Start Command:**
   ```
   cd backend && gunicorn -w 4 -b 0.0.0.0:5000 "certivault:create_app()"
   ```
   *(Uses Gunicorn production WSGI server for reliability and performance)*
6. **Environment Variables:**
   - `FLASK_ENV` = `production`
   - `FLASK_DEBUG` = `False`
   - `SECRET_KEY` = (Render generates automatically)

7. **Deploy** → Wait 2-3 minutes

Note backend URL (e.g., `https://certivault-backend.onrender.com`)

#### Frontend Deploy

1. Create new Web Service on Render
2. **Name:** `certivault-frontend`
3. **Environment:** Node
4. **Build Command:**
   ```
   cd frontend && npm install && npm run build
   ```
5. **Start Command:**
   ```
   cd frontend && npm run start
   ```
6. **Environment Variables:**
   - `NEXT_PUBLIC_API_URL` = `https://certivault-backend.onrender.com` (from backend)

7. **Deploy** → Wait 3-5 minutes

## Important Notes

### Free Tier Limits

- Services spin down after 15 min of inactivity (one free service per account)
- First request after inactivity = ~10 second cold start
- Data resets if service is redeployed
- Perfect for testing, not production

### Database Storage

Currently using SQLite. When service redeploys:
- **All user data is lost** (SQLite only works locally)
- To keep data, upgrade to PostgreSQL (paid)

For MVP/testing: SQL works fine. For production: add PostgreSQL.

### CORS Configuration

Frontend and backend are separate domains. Already configured in Flask for this setup. If you get CORS errors:

Edit `backend/certivault/routes.py`:
```python
# Already set up, but verify:
@app.after_request
def after_request(response):
    response.headers.add('Access-Control-Allow-Origin', '*')
    response.headers.add('Access-Control-Allow-Methods', 'GET,POST,PATCH,DELETE')
    response.headers.add('Access-Control-Allow-Headers', 'Content-Type,Authorization')
    return response
```

## Monitoring & Debugging

### View Logs

1. Go to service on Render dashboard
2. Click "Logs" tab
3. See real-time output

### Common Issues

**"Service won't start"**
- Check logs for error messages
- Verify Build/Start commands match exactly
- Ensure `requirements.txt` and `package.json` exist

**"Frontend can't connect to backend"**
- Verify `NEXT_PUBLIC_API_URL` has correct backend URL
- Check CORS is enabled in Flask backend
- Frontend redeploy may be needed after backend URL changes

**"Page takes 10+ seconds to load"**
- First request is cold start on free tier
- Normal behavior, happens after 15 min inactivity
- Subsequent requests faster

## Upgrading to Production

When ready for production:

1. **Add PostgreSQL** from Render marketplace
   - Keeps data persistent
   - Costs $7/month
   
2. **Use custom domain**
   - Add your domain to Render settings
   - Free SSL certificate included

3. **Monitor performance**
   - Render dashboard shows metrics
   - Scale if needed
